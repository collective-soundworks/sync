getTimeFunction() → {Number}

Source:
  • server/index.js, line 5
Returns:

monotonic, ever increasing, time in second. When possible the server code should define its own origin (i.e. time=0) in order to maximize the resolution of the clock for a long period of time. When SyncServer~start is called the clock should be running (cf. audioContext.currentTime that needs user interaction to start)

Type
Number
Example
const startTime = process.hrtime();

const getTimeFunction = () => {
  const now = process.hrtime(startTime);
  return now[0] + now[1] * 1e-9;
};
